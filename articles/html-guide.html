ALEGATOR1209 FictAdvisor is capable to parse, process and display Markdown code.

1. Creating article file

All articles must be stored in the articles folder of this repository. No nested directories are allowed as they will be ignored during the build.

File name must be concise, unique (within articles folder) and end with .html marking that it's an HTML file.

Be careful when choosing filename because it will be used as a path to the article (with .html ending trimmed).

2. Metadata

At the start of the file a metadata section should be placed. That's where article title and authors are.

Format of the metadata section is following: 
            
<head>
    <title>Article title</title>
    <comment>Article author</comment>
</head>

3. Content

Content of the article can be composed of the usual HTML tags. For the most common, there are predefined styles.
  • Use <h1>, <h2>, <h3>, etc for headers. Automatic anchoring isn't supported yet, so you should manually assign headers id attribute if you want it.
  • Use <img> for images and always give the full URL to the href parameter.
  • Use <ul>, <ol> and <li> tags for lists.
  • Use <a> for links. Always pass the full URL.
  • Use <em>, <b>, <i> for text formatting.
  • Use <br/> for line breaks.
  • Use <code> for single-line code samples and <pre><code>...</code></pre> for multiline ones.
  • Use any other HTML tags and even combine them with JS or CSS if needed, but we can't guarantee pretty styling for them.

4. Page management

All articles are listed at the articles page. Two special files are present to alter this behavior.
Article can be included into the featured articles list that is displayed at the Fict Advisor front page by including it into __CHOSEN__ file. Articles, included in the __HIDDEN__ file aren't displayed at the /articles page, but still can be accessed with the direct link. It's useful for drafts. In both cases, you should include articles in both files using its pathname (filename without extension) separated with the line break. For example, to list articles in files sample1.html and sample2.md as featured, you should add following lines to the __CHOSEN__ file: 

sample1
sample2
